# Бизнес-постановка

Нужно добавить на сайт интернет-магазина персональную страницу поздравления пользователя с днем рождения.

Цель страницы — поздравить пользователя, выдать ему персональный подарочный купон и показать подборку товаров, которые могут быть ему интересны. Страница должна повышать возвращаемость пользователя в день рождения и мотивировать его сделать покупку с купоном.

# Пользовательский сценарий

1. Пользователь авторизован на сайте.
2. В день рождения пользователь получает email/push/баннер на главной странице.
3. Пользователь переходит на страницу поздравления.
4. Система показывает персональное поздравление по имени.
5. Система показывает подарочный купон.
6. Пользователь может скопировать купон или применить его к корзине.
7. Ниже пользователь видит подборку товаров, которые могут его заинтересовать.
8. Пользователь переходит в карточку товара или добавляет товар в корзину.

# Макеты словами

Страница состоит из четырех блоков:

## Верхний поздравительный блок

Большой заголовок: "Анна, поздравляем с днем рождения!".

Под заголовком короткий текст: "Дарим вам персональный купон и подобрали товары, которые могут понравиться".

Фон праздничный, но не перегруженный. В мобильной версии блок занимает первый экран частично, чтобы ниже был виден купон.

## Блок купона

Карточка с купоном:

- размер скидки, например "15%";
- промокод, например "BDAY-ANNA-15";
- срок действия, например "действует 7 дней";
- кнопка "Скопировать";
- кнопка "Применить в корзине".

Если купон уже использован, вместо кнопок показывается текст "Купон уже применен к заказу".

Если купон истек, показывается текст "Срок действия купона закончился".

## Блок товарных рекомендаций

Сетка товаров:

- изображение товара;
- название;
- цена;
- старая цена, если есть скидка;
- рейтинг;
- кнопка "В корзину".

В desktop версии показываем 4 товара в ряд, в mobile — горизонтальный скролл или 2 товара в ряд.

Рекомендации должны учитывать историю покупок, просмотры, избранное и категорию последней покупки. Если персональных рекомендаций нет, показывать популярные товары в категориях пользователя.

## Нижний блок

Короткий текст: "Подарок ждет вас до конца недели".

Кнопка "Перейти в каталог".

# Состояния

- loading — идет загрузка профиля пользователя, купона и рекомендаций;
- success — все данные загружены;
- noBirthday — сегодня не день рождения пользователя, страница недоступна;
- unauthorized — пользователь не авторизован;
- couponUsed — купон уже использован;
- couponExpired — купон истек;
- recommendationsEmpty — персональных рекомендаций нет, показываем fallback-подборку;
- error — не удалось загрузить данные.

# Ограничения

- Купон должен быть персональным и не должен применяться к чужому аккаунту.
- Один пользователь может использовать birthday-купон только один раз в год.
- Нельзя показывать страницу неавторизованному пользователю.
- Нельзя показывать купон, если дата рождения пользователя не подтверждена.
- Нужно защититься от ручного перебора промокодов.
- Купон должен применяться в корзине только при выполнении условий акции.
- Если сервис рекомендаций недоступен, страница должна работать с fallback-подборкой.

# Аналитика

Нужно логировать события:

- birthday_page_opened;
- birthday_coupon_copied;
- birthday_coupon_applied;
- birthday_recommendation_clicked;
- birthday_product_added_to_cart;
- birthday_page_error.

Для событий нужны параметры:

- user_id;
- coupon_id;
- recommendation_strategy;
- product_id;
- source: email, push, homepage_banner, direct.

# Платформы

В первую очередь нужна web-версия сайта.

iOS и Android приложения пока не входят в первый релиз, но API нужно проектировать так, чтобы мобильные приложения могли позже использовать тот же endpoint для поздравления, купона и рекомендаций.
